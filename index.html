// WHAT IS AMAZON S3?

Dokumentasi & Spesifikasi Core Cloud Storage

💡 APA ITU AMAZON S3?

Amazon S3 (Simple Storage Service) adalah layanan Object Storage berbasis cloud yang dikelola penuh oleh AWS. Layanan ini dirancang untuk menyimpan, mengamankan, dan mengambil data dalam jumlah berapa pun dari mana saja di internet.

Berbeda dengan sistem file tradisional (Block Storage), S3 menyimpan data sebagai Objek di dalam wadah yang disebut Bucket.

⚙️ CORE SPECIFICATIONS

📦 Ukuran Objek (Object Size)

  • Ukuran Minimum: 0 Bytes.
  • Ukuran Maksimum: 5 Terabytes (TB) untuk satu objek tunggal.
  • Batas Sekali Upload: Operasi tunggal PUT maksimal menerima 5 Gigabytes (GB). Untuk file di atas 5 GB, wajib menggunakan Multipart Upload API.

🔑 Apa itu "Key"?

  • Key adalah nama unik atau pengenal (identifier) global bagi sebuah objek di dalam sebuah Bucket.
  • Setiap objek di dalam S3 dapat diakses secara unik melalui kombinasi dari:
    Bucket Name + Object Key + Version ID.

🛡️ DURABILITY

99.999999999%

(11 Sembilan / 11 Nines)

Didesain untuk memberikan ketahanan data yang ekstrem dengan mereplikasi objek secara otomatis ke minimal 3 Availability Zones (AZ) yang terpisah di dalam satu Region.

🌐 AVAILABILITY SLA

Tingkat ketersediaan operasional (uptime) dijamin oleh AWS melalui Service Level Agreement (SLA) yang bervariasi bergantung tipe kelas penyimpanan:

  • S3 Standard: SLA Komitmen > 99.9% (Didesain untuk 99.99% ketersediaan).
  • S3 Standard-IA: SLA Komitmen > 99.0%.

🗂️ STORAGE CLASSES & ACCESS METHOD

⚡ Jenis-Jenis Penyimpanan (Storage Classes):

  • S3 Standard: Akses cepat, frekuensi tinggi (default).
  • S3 Intelligent-Tiering: Otomatis menghemat biaya berdasarkan pola akses data.
  • S3 Standard-IA: Untuk data jarang diakses tapi butuh cepat saat diperlukan.
  • S3 One Zone-IA: Data jarang diakses dan disimpan hanya di 1 AZ (lebih murah).
  • S3 Glacier Instant Retrieval: Arsip dengan akses hitungan milidetik.
  • S3 Glacier Flexible Retrieval: Arsip berkala (akses 1 menit hingga 5 jam).
  • S3 Glacier Deep Archive: Arsip jangka panjang (akses 12 jam, paling murah).

🛠️ Bagaimana Menggunakan & Mengakses S3?

Amazon S3 dapat digunakan dengan membuat Bucket terlebih dahulu, lalu mengunggah file ke dalamnya. Kamu bisa mengelolanya via:

  • AWS Management Console (Antarmuka Web), AWS CLI (Terminal/Command Line), dan AWS SDK (Kode Program).

🌐 Protokol Akses Internet

Seluruh objek di Amazon S3 diekspos sebagai RESTful endpoint. Protokol standar internet yang digunakan untuk berinteraksi, mengirim (upload), dan mengambil (download) objek adalah HTTP dan HTTPS.
